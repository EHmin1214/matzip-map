# 디자인 인수인계서 — #1 시각적 계층 개선

---

## 디자인 철학

이 서비스의 핵심 정체성은 **"나만의 맛집 노트"**입니다.

- SNS이지만 전시/노출이 목적이 아닌 **개인 아카이브**에 가까운 공간
- 기능적 아쉬움 없이, 동시에 **미니멀하고 여유로운** 톤 유지
- 사용자가 **자기만의 공간을 소유하고 있다는 느낌**이 핵심
- 레퍼런스 톤: Are.na, Things 3, Notion 개인 페이지

현재 컬러(`#655d54` 웜브라운)와 서체(Noto Serif + Manrope)는 이 방향에 잘 맞고 있으므로 유지합니다. 이번 수정은 **밀도를 줄이고 여백을 확보**하여 디자인 톤에 맞는 여유로움을 만드는 작업입니다.

---

## 수정 1: 데스크톱 사이드바 여백 확보

**파일:** `frontend/src/components/Sidebar.js` (또는 사이드바 담당 컴포넌트)

**현재 문제:** 장소 목록, 팔로잉 섹션, 프로필 카드가 300px 안에 빽빽하게 들어가 있어 여유로운 톤과 맞지 않음

**변경 사항:**

| 속성 | 현재 (추정) | 변경 |
|------|-------------|------|
| 섹션 간 간격 (margin/gap) | ~16-24px | **32px** |
| 사이드바 내부 좌우 패딩 | ~16px | **20px** |
| 섹션 구분선 (border-bottom 등) | 있음 | **제거** — 여백만으로 구분 |

구분선을 없애고 여백을 넓히면 "줄 그어 나눈 목록"이 아니라 "여유 있게 배치된 개인 공간" 느낌이 됩니다.

---

## 수정 2: 팔로잉 섹션 아코디언(접기/펼치기)

**파일:** 사이드바 내 팔로잉 유저 체크박스 리스트 영역

**현재 문제:** 팔로잉 유저가 많아지면 사이드바를 과도하게 차지하여 "내 장소"가 밀려남

**변경 사항:**

- 팔로잉 섹션을 **collapsible 아코디언**으로 변경
- **기본 상태: 접혀 있음**
  - 섹션 헤더("팔로잉" 텍스트 + 유저 수 뱃지) + 우측에 펼침 화살표 아이콘(`expand_more` / `expand_less`)만 표시
  - 한 줄 높이만 차지
- **펼친 상태:**
  - 헤더 클릭 시 체크박스 리스트가 아래로 펼쳐짐
  - 부드러운 전환 애니메이션 (height transition, 0.2s ease)

**의도:** "내 장소 목록"이 사이드바의 주인공이 되고, 팔로잉은 필요할 때만 꺼내 보는 보조 기능으로 위계를 명확히 함

---

## 수정 3: 사이드바 장소 검색 입력란 축소

**파일:** 사이드바 내 검색 입력 영역

**현재 문제:** 검색 입력란이 항상 펼쳐져 있어 사용 빈도 대비 공간을 많이 차지함

**변경 사항:**

- **기본 상태:** 돋보기 아이콘(`search`)만 표시 (한 줄, 아이콘 버튼)
- **클릭 시:** 입력란이 옆으로 확장되며 텍스트 입력 가능 (width transition, 0.2s ease)
- **포커스 해제 + 입력값 없음:** 다시 아이콘만 표시로 축소
- **포커스 해제 + 입력값 있음:** 입력란 확장 상태 유지 (검색어 보존)

## 수정 4: 모바일 카드 간격 확보

**파일:** `FeedTab.js`, `SearchTab.js` (카드 리스트 영역)

**현재 문제:** 카드가 촘촘하게 붙어 있으면 각 장소의 경계가 불명확하고 시각적으로 피로함

**변경 사항:**

| 속성 | 현재 (추정) | 변경 |
|------|-------------|------|
| 카드 간 간격 (gap/margin) | ~8-12px | **16px** |
| 카드 배경색 | 확인 필요 | **#ffffff** |
| 리스트 배경색 | 확인 필요 | **#faf9f6** |

카드(`#fff`)와 배경(`#faf9f6`) 간 미세한 색상 차이 + 넉넉한 간격으로 각 카드가 독립된 단위로 인식되게 합니다. 이미 이렇게 되어 있다면 간격만 조정하면 됩니다.

---

## 수정 5: 모바일 바텀시트 딤 오버레이

**파일:** `RestaurantPanel.js` (또는 바텀시트 래퍼)

**현재 문제:** 바텀시트가 올라왔을 때 뒤의 지도와 시각적으로 레이어가 분리되지 않아 공간감이 부족함

**변경 사항:**

- RestaurantPanel이 열릴 때 뒤쪽에 **반투명 딤 오버레이** 추가
- 스타일: `background: rgba(47, 52, 48, 0.12)` (기존 텍스트 컬러 `#2f3430` 기반, 약하게)
- 시트가 열릴 때 fade-in (opacity 0→1, 0.2s ease)
- 딤 영역 클릭 시 시트 닫힘
- peek 모드(210px)에서는 딤 없이, **full 모드에서만** 딤 적용

**주의:** 딤이 너무 강하면 지도 확인이 어려우므로 opacity 0.12~0.15 정도로 은은하게

## 수정하지 않는 것

- 사이드바 너비(300px) 유지
- 컬러 팔레트, 서체, 아이콘 시스템 변경 없음
- 컴포넌트 구조 변경 없음 (스타일링만 수정)
