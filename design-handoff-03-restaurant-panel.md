# 디자인 인수인계서 — #3 모바일 바텀시트 peek 모드 정보 노출

---

## 디자인 방향

모바일 바텀시트(RestaurantPanel)의 peek 모드에서 유저가 "더 보고 싶다"는 동기를 느낄 수 있도록, 핵심 정보를 한 줄로 미리 보여줍니다.

---

## 현재 상태 (수정 불필요 — 확인 완료)

- **사진 dot indicator:** 이미 구현됨 (5px 흰색 도트, 하단 중앙)
- **공유/길찾기 버튼:** 이미 "지도"/"공유" 버튼 → 드롭다운 방식으로 잘 정리되어 있음

---

## 수정: peek 모드 미리보기 정보 추가

**파일:** `frontend/src/components/RestaurantPanel.jsx`

**현재 문제:** peek 모드(210px or 160px)에서는 장소 이름 + 상태/별점 + 갤러리까지만 보이고, 좋아요 수 등 인터랙션 정보가 잘려서 "위로 올려 더 보고 싶다"는 동기가 약함

**변경 사항:**

peek 모드에서 장소 이름 바로 옆 또는 아래에 **좋아요 수 + 댓글 수**를 한 줄 요약으로 표시

- 위치: 장소 이름(`h2`) 우측 또는 상태 블록(`StatusBlock`) 바로 아래
- 스타일:
  ```
  display: inline-flex
  alignItems: center
  gap: 8px
  fontSize: 11px
  fontFamily: FL (Manrope)
  color: C.outlineVariant (#afb3ae)
  ```
- 내용 예시: `♡ 3  💬 2` (하트 아이콘 + 숫자, 말풍선 아이콘 + 숫자)
- Material Symbols 사용: `favorite` (좋아요), `chat_bubble` (댓글)
- 좋아요/댓글 0개면 해당 항목 숨김
- 둘 다 0개면 이 줄 자체를 렌더하지 않음

**구현 방식:**

이 요약 줄은 peek/full 모두에서 보여도 됩니다 (full에서는 아래에 상세 SocialBlock이 있으므로 중복이지만, 간결한 요약이라 거슬리지 않음). 만약 중복이 신경 쓰인다면 `sheetMode === "peek"`일 때만 렌더해도 좋습니다.

**시각적 결과 (peek 모드):**

```
━━━━━━━━ (핸들) ━━━━━━━━

장소 이름                    ✕
🔖 가고싶다 · ⭐ 4.5
♡ 3  💬 2                    ← 추가
┌─────────────────────┐
│     [사진 갤러리]      │
│       · · ● · ·       │
└─────────────────────┘
        ︿ (더보기 화살표)
```
